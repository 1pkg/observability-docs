

By default, the Node.js agent will filter common sensitive information before sending errors and metrics to the Elastic APM server.

It's possible for you to tweak these defaults or remove any information you don't want to send to Elastic APM:

* By default, the Node.js agent will not log the body of HTTP requests.
    To enable this,
    use the <DocLink id="serverlessObservabilityApmAgentsNodejsConfigure" section="capturebody">`captureBody`</DocLink> config option

* By default, the Node.js agent will filter certain HTTP headers known to contain sensitive information.
    To disable this,
    use the <DocLink id="serverlessObservabilityApmAgentsNodejsConfigure" section="sanitizefieldnames">`sanitizeFieldNames`</DocLink> config option
