

When an <DocLink id="serverlessObservabilityApmApiKeys">API key</DocLink> or <DocLink id="serverlessObservabilityApmSecretToken">secret token</DocLink> is configured,
anonymous authentication must be enabled to collect RUM data.
To enable anonymous access, set either <DocBadge><DocIcon size="s" type="unlink" title="missing link"/> missing link</DocBadge>{/*  <DocLink id="enApmGuideConfigurationRum" section="enable-rum">`apm-server.rum.enabled`</DocLink> */} or
 <DocBadge><DocIcon size="s" type="unlink" title="missing link"/> missing link</DocBadge>{/* <DocLink id="enApmGuideConfigurationAnonymous" section="anonymous-agent-access">`apm-server.auth.anonymous.enabled`</DocLink> */} to `true`.

Because anyone can send anonymous events to the APM Server,
additional configuration variables are available to rate limit the number anonymous events the APM Server processes;
throughput is equal to the `rate_limit.ip_limit` times the `rate_limit.event_limit`.

See <DocBadge><DocIcon size="s" type="unlink" title="missing link"/> missing link</DocBadge>{/*  <DocLink id="enApmGuideConfigurationAnonymous">Configure anonymous authentication</DocLink> */} for a complete list of options and a sample configuration file.
