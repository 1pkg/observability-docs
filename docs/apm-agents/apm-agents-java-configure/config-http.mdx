### `capture_body_content_types` (<DocBadgeAdded tipContent="Added in 1.5.0">1.5.0</DocBadgeAdded> performance)

Configures which content types should be recorded.

The defaults end with a wildcard so that content types like `text/plain; charset=utf-8` are captured as well.

This option supports the wildcard `*`, which matches zero or more characters.
Examples: `/foo/*/bar/*/baz*`, `*foo*`.
Matching is case insensitive by default.
Prepending an element with `(?-i)` makes the matching case sensitive.

### `use_path_as_transaction_name` (<DocBadgeAdded tipTitle="Added in 1.0.0" tipContent="Changing this value at runtime is possible since version 1.22.0">1.0.0</DocBadgeAdded>)

If set to `true`,
transaction names of unsupported or partially-supported frameworks will be in the form of `$method $path` instead of just `$method unknown route`.

<DocCallOut title="Warning" color="warning">
If your URLs contain path parameters like `/user/$userId`,
you should be very careful when enabling this flag,
as it can lead to an explosion of transaction groups.
Take a look at the <DocLink id="serverlessObservabilityApmAgentsJavaConfigure" section="transaction_name_groups-added[1330]">`transaction_name_groups`</DocLink> option on how to mitigate this problem by grouping URLs together.
</DocCallOut>
