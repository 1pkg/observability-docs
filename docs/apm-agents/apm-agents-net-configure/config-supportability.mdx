### `LogLevel`

<DocLink id="serverlessObservabilityApmAgentsNetConfigure" section="dynamic-configuration"><DocBadge>Dynamic</DocBadge> </DocLink>

Sets the logging level for the agent.

Valid options: `Critical`, `Error`, `Warning`, `Info`, `Debug`, `Trace` and `None` (`None` disables the logging).

<DocCallOut title="Important" color="warning">
The `UseElasticApm()` extension offers an overload to pass an `IConfiguration` instance to the agent.
When configuring your agent in this way, as is typical in an ASP.NET Core application,
you must instead set the `LogLevel` for the internal APM logger under the `Logging` section of `appsettings.json`. More details, including a <DocBadge><DocIcon size="s" type="unlink" title="missing link"/> missing link</DocBadge>{/*  <DocLink id="enApmAgentDotnetConfigurationOnAspNetCore" section="sample-configuration-file">sample configuration file</DocLink> */} are available in <DocBadge><DocIcon size="s" type="unlink" title="missing link"/> missing link</DocBadge>{/*  <DocLink id="enApmAgentDotnetConfigurationOnAspNetCore">Configuration on ASP.NET Core</DocLink> */}.
</DocCallOut>

| Environment variable name | IConfiguration or Web.config key |
|---|---|
| `ELASTIC_APM_LOG_LEVEL`   | `ElasticApm:LogLevel` |

| Default | Type |
|---|---|
| `Error`                 | String |
